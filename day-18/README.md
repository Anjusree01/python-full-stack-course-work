instance of a class -> object
set of variables and functions is called blueprint
class is a blueprint. It is a combination of attributes(variables) and methods(functions)

binding and restricting the data/information -> encapsulation
__password -> private #can be accessed only inside the class
_post -> protected #can be accessed inside the class
followers -> public #can be acccessed inside and outside the class


the process of extracting features from parent class to child class -> inheritance

same method different actions -> polymorphism

hiding the complexity -> abstraction

attribute- properties or characteristics of an object
types of attributes:
class attribute #the variables that depends on the class(application)
instance attribute #the variables that depends on the object(user)

methods- functions or actions that the object can perform
types of methods:
class method #the method that deals with the class attribute
instance method #the method that deals with the instance attribute
static method #apart of class and instance we use this, helper method

self-refers to the current object
